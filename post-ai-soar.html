AI‑SOAR + Wazuh: From Toy Demo to Real SOC Powerhouse

Joel Yang
Security Engineer
Oct 8, 2025
4 min read

With Wazuh as your telemetry + response backbone and Tracecat as the workflow brain, you get a SOC copilot that actually ships. This post turns a simple POC into a blueprint you can deploy.

Why This Matters

  • Auto-triage: LLMs summarize noisy findings into crisp next steps.
  • Enrichment on tap: CVE, package age, vendor advisories, TI pivots.
  • Decisioning: "Is this risky enough to wake someone up?" → scored.
  • Action: Kick Active Response, open a case, ping Slack/Jira/Discord.

What We Built (POC Recap)

Goal: Pull Syscollector → Packages for an agent, ask AI to rank risk, then ship a compact case/alert.

Stack: Wazuh Manager (telemetry), Tracecat (AI-SOAR), OpenAI gpt-4o-mini (reasoning), Discord (demo output).

Step 1: Get a Wazuh JWT Token

tracecat-action: get_manager_token 
# ACTION: get_manager_token
function: core.http_request
with:
  method: GET
  url: https://<wazuh-manager>:55000/security/user/authenticate?raw=true
  auth:
    username: wazuh
    password: <password>
  verify_ssl: false
  timeout: 20

Step 2: Extract the Token

tracecat-action: extract_token 
# ACTION: extract_token
function: core.transform.reshape
with:
  value: ${{ ACTIONS.get_manager_token.result.data }}

Step 3: Pull Syscollector Packages

tracecat-action: get_agent_packages 
# ACTION: get_agent_packages
function: core.http_request
with:
  url: https://<wazuh-manager>:55000/syscollector/<AGENT_ID>/packages
  method: GET
  headers:
    Authorization: Bearer ${{ ACTIONS.extract_token.result }}
    Content-Type: application/json
  verify_ssl: false

Tip: Use the Manager API when you can. If you need raw search power across alerts, hit the Indexer via core.http_request until a native OpenSearch starter kit lands.

Step 4: AI Analysis → Structured Case Payload

Feed packages into the AI for risk-ranked, schema-constrained output:

tracecat-action: core.ai.agent 
# ACTION: core.ai.agent
with:
  model_provider: openai
  model_name: gpt-4o-mini
  user_prompt: |
    You are a SOC analyst. Input is Wazuh Syscollector
    "packages" JSON for one agent.
    Output ONLY valid JSON with this exact shape:
    {
      "summary": "Short human-readable overview (<= 6 lines)",
      "case": {
        "title": "Recommended case title",
        "severity": "low|medium|high|critical",
        "tags": ["wazuh", "packages", "syscollector"],
        "comment": "Top risky/outdated packages + remediation"
      }
    }

Step 5: Ship It

tracecat-action: post_discord 
# ACTION: post_discord
function: core.http_request
with:
  method: POST
  url: https://discord.com/api/webhooks/<id>/<token>
  headers:
    Content-Type: application/json
  json:
    content: ${{ ACTIONS.ai_package_review.result.summary }}

In production, point that payload at Jira/ServiceNow, create a case in TheHive, or invoke Wazuh Active Response.

Production-Grade Upgrades

  • Role-based guardrails — split read vs response; gate high-blast actions behind humans.
  • Prompt hardening — enforce strict JSON + schema; add few-shot examples.
  • Idempotency + retries — use correlation IDs, cache tokens, backoff on failures.
  • Observability — emit metrics (latency, token usage), log full I/O to secure storage.
  • Secrets & compliance — creds in KMS; scrub PII before AI calls.

"Tracecat makes the wiring easy; Wazuh brings reach and muscle. Add a disciplined AI layer and you've got a SOC copilot that reduces noise, speeds response, and documents everything without drama."
