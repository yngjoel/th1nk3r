Step 1: Install InfluxDB V2

Add the InfluxData repository and install InfluxDB 2.0 on Debian/Ubuntu:

install_influxdb.sh 
# Add InfluxData GPG key and repository
curl -s https://repos.influxdata.com/influxdata-archive.key | \
  gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/influxdata.gpg
echo "deb https://repos.influxdata.com/debian stable main" | \
  sudo tee /etc/apt/sources.list.d/influxdata.list

sudo apt-get update
sudo apt-get install -y influxdb2

Step 2: Install & Configure Telegraf

Telegraf is the data collection agent. Install it and create a custom configuration for monitoring Wazuh:

install_telegraf.sh 
sudo apt-get install -y telegraf

Custom Wazuh AIO Configuration

Here's a comprehensive telegraf.conf that monitors the Wazuh manager, dashboard, indexer, plus system metrics:

/etc/telegraf/telegraf.conf 
# Monitor ossec.log for Wazuh Manager activity
[[inputs.tail]]
  files = ["/var/ossec/logs/ossec.log"]
  from_beginning = false

# Monitor Wazuh services via systemd
[[inputs.systemd_units]]
  unittype = "service"
  units = ["wazuh-manager.service",
           "wazuh-dashboard.service",
           "wazuh-indexer.service"]

# Monitor Wazuh processes
[[inputs.procstat]]
  pattern = "wazuh|ossec"

# Check indexer cluster health
[[inputs.http]]
  urls = ["https://localhost:9200/_cluster/health"]
  method = "GET"
  insecure_skip_verify = true

# System metrics
[[inputs.mem]]
[[inputs.disk]]
[[inputs.nstat]]

# Output to InfluxDB v2
[[outputs.influxdb_v2]]
  urls = ["http://localhost:8086"]
  token = "$INFLUX_TOKEN"
  organization = "your-org"
  bucket = "wazuh-monitoring"

Step 3: Grafana Visualization

Install Grafana and connect it to InfluxDB using the Flux query language:

install_grafana.sh 
sudo apt-get install -y grafana
sudo systemctl enable --now grafana-server

Once Grafana is running, add InfluxDB as a data source (using Flux), then build dashboards that show service health, memory usage, disk I/O, and ossec.log activity â€” all in real time.
